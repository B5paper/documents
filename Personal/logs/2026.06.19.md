* vim 中我想使用 vnoremap <leader>b 对选中的文件左右各添加两个星号 **，

    vnoremap <leader>b mboma<esc>`ai**<esc>`ba**<esc> 为什么不行？正确方法是什么？

    你原来的思路非常有趣，通过标记（marks）来定位选区的两端是一个很极客的想法。

    不过，这个映射之所以无法如愿工作，主要是因为 **Vim 模式切换的特性** 以及 **光标位置的微妙变化**。

    ---

    ### 为什么你原来的方案不行？

    我们来拆解一下你的命令 `mboma<esc>`ai`ba<esc>`：

    1. **`mboma` 的逻辑死胡同**：在 Visual 模式下，当你按下 `m` 时，Vim 会**立刻退出 Visual 模式**并把当前光标所在的那*一个*字符标记为 `b`。接着你按 `o`（在正常模式下是向下新开一行），然后又按 `m`、`a`。这时候你原本的选区早就消失了，光标也已经跑偏了。
    2. **光标位置会发生偏移**：即使你在 Normal 模式下手动用 ``a` 和 ``b` 找到了位置，当你插入左侧的 `` 后，右侧标记 ``b` 的相对字符位置就会向后偏移 2 个字符。这时候再直接跳到 ``b`，加星号的位置就会错位（变成加在倒数第二个字符前面）。

    ---

    ### 正确且高效的方法

    在 Vim 中，处理选区文本最地道、最常用的方法有两种：

    #### 方法一：利用寄存器（最推荐，简单且丝滑）

    按 `c` 剪切选区，然后利用 `<C-r>"` 瞬间把剪切的内容连同星号一起塞回去。

    ```vim
    vnoremap <leader>b c**<C-r>"**<esc>

    ```

    * **原理解析**：
    * `c`：删除选中的文本，并进入插入模式（此时选中的文本自动存入了默认寄存器 `"` ）。
    * ``：输入左边的星号。
    * `<C-r>"`：在插入模式下，直接粘贴默认寄存器里的文本。
    * ``：输入右边的星号。
    * `<esc>`：返回正常模式。



    #### 方法二：利用内置的选区标记 `<` 和 `>`

    如果你非要用类似“标记跳转”的思路，Vim 其实自带了代表选区开头和结尾的内置标记：`<` 和 `>`。

    ```vim
    vnoremap <leader>b <esc>`.>a**<esc>`.`<i**<esc>

    ```

    * **原理解析**：必须先用 `<esc>` 退出可视模式，然后利用 ``>`（选区末尾）和 ``<`（选区开头）进行精准定位。为了防止先加左边导致右边偏移，这里**先加右边，再加左边**。

    ---

    ### 方法一：针对“部分单词”或单行内的选区（最常用）

    如果你只是在某一行里用 `v` 选中了几个词，想给它们前后加 ``，可以利用正则表达式的 `\%V` 匹配元字符（它代表“仅匹配选区内的内容”）：

    ```vim
    vnoremap <leader>b :s/\%V.*\%V/**&**/<CR>

    ```

    * **原理解析**：
    * `:`：进入命令行模式，Vim 自动补全为 `:'<,'>`（代表选区范围）。
    * `s/`：开始替换。
    * `\%V.*\%V`：极其关键！`\%V` 告诉 Vim 匹配必须在选区内开始，也在选区内结束。`.*` 匹配选区里的所有文本。
    * `/&/`：`&` 代表刚才匹配到的原文本，我们在它左右各加上 ``。
    * `<CR>`：敲回车执行。



    ---

    ### 方法二：针对“多行”或“整行”的选区

    如果你是用 `V`（行可视模式）选中了一整行或多行，想让这几行的最前端和最后端加上 ``，可以使用下面这个更简单的正则：

    ```vim
    vnoremap <leader>b :s/^.\+$/\*\*&\*\*/<CR>

    ```

    或者直接用替代符（省去转义）：

    ```vim
    vnoremap <leader>b :s#.*#**&**#<CR>

    ```

    * **注意**：这种方法会把选中的**每一行**都分别用 `` 包裹起来。

* 如何删除 windows 所复制的内容里的大量 ^M

    你在远程 Linux/Cygwin 环境下看到的大量 `^M`，是因为 Windows 和 Linux 对“换行符”的定义不同。Windows 使用 `CRLF`（`\r\n`，`^M` 就是 `\r`），而 Linux 只使用 `LF`（`\n`）。

    当你把 Windows 的文本复制粘贴到远程时，这些 `^M` 就被一起带过来了。这里有几种高效的删除方法，取决于你是想在 **Vim 内** 还是 **命令行中** 处理。

    ---

    ### 方法一：在 Vim 中直接删除（最推荐）

    如果这些 `^M` 已经粘贴进了 Vim 编辑器里，你可以用替换命令秒杀它们。

    #### 1. 终极替换命令

    在 Vim 的普通模式下，输入以下命令并回车：

    ```vim
    :%s/\r//g

    ```

    > **注意**：这里的 `\r` 在 Vim 内部代表的就是 `^M`。执行后，全篇的 `^M` 都会被清空。

    #### 2. 原生快捷键敲击法（如果上面的不行）

    有时候 `\r` 匹配不到，你需要用特殊的按键组合来输入真正的 `^M`：

    1. 输入 `:%s/`
    2. 依次按下快捷键 **`Ctrl + V`**，然后按下 **`Ctrl + M`**（此时屏幕上会出现一个高亮的 `^M`）
    3. 继续输入 `//g`
    4. 完整的命令看起来是：`:%s/^M//g`，回车即可。

    #### 3. 改变文件格式法

    如果你的 Vim 识别到了这是个 Windows 文件，你也可以强制让 Vim 把它转成 Unix 格式，`^M` 会自动消失：

    ```vim
    :set fileformat=unix
    :w

    ```

    ---

    ### 方法二：在 Linux 命令行处理（针对已有文件）

    如果你是把内容粘成了一个文件（比如 `test.txt`），可以用命令行工具直接清洗文件：

    #### 1. 使用 `dos2unix` 工具（最省心）

    Cygwin 和绝大多数 Linux 都有这个工具。如果没有，可以先安装。

    ```bash
    dos2unix filename.txt

    ```

    这会直接修改文件，把所有的 `CRLF` 自动转换为 `LF`。

    #### 2. 使用 `sed` 命令（万能自带）

    如果不想安装额外工具，直接用 `sed` 流编辑器：

    ```bash
    sed -i 's/\r//g' filename.txt

    ```

    既然你在用 Vim，下次从 Windows 复制了文本，准备在远程 Vim 里鼠标右键粘贴前，**先在 Vim 里开启粘贴模式**：

    ```vim
    :set paste

    ```

    进入 `INSERT (paste)` 状态后再右键粘贴。这样 Vim 会以纯文本流接收，通常能极大地减少因为终端自动缩进或格式转换产生的奇奇怪怪的符号（粘贴完后记得输入 `:set nopaste` 恢复正常编辑）。

* 能否设置一个映射，比如 <leader>p，使得先 :set paste 再 "+p，最后再 :set nopaste 恢复原样？

    可以，而且这是非常经典的 Vim 优化操作！

    你可以将以下配置加入到你的配置中（如果是远程服务器，加到远程的 `~/.vimrc` 里；如果是本地，加到 Cygwin 的 `~/.vimrc` 里）。

    不过这里有一个**核心细节**需要先明确：

    > 正如我们在前两轮对话中讨论的，如果你的远程 SSH 会话中 `"+p` 依然是**失效（不可用）**的状态，那么即便做了这个映射，`"+p` 那一步依然无法从 Windows 剪贴板读取数据。
    > 这个映射生效的前提是：你已经通过**方法一（开启 X11 转发）**修复了 `"+` 寄存器，或者你是在 **Cygwin 本地** 的 Vim 里使用它。

    ### 1. 针对 `"+p` 的快捷键映射（前提：剪贴板已修复）

    将以下代码写入 `~/.vimrc`：

    ```vim
    " 使用 <Leader>p 自动切换粘贴模式并从系统剪贴板粘贴
    nnoremap <Leader>p :set paste<CR>"+p:set nopaste<CR>

    ```

    * `nnoremap`：表示在普通模式（Normal mode）下生效，且不进行递归映射。
    * `<CR>`：代表回车键（Enter），用来执行前面的命令。
    * **运行流程**：按下 `<Leader>p` $\rightarrow$ 执行 `:set paste` $\rightarrow$ 执行 `"+p` 粘贴 $\rightarrow$ 执行 `:set nopaste` 恢复。

    ---

    ### 2. 替代方案：如果 `"+p` 依然失效，如何用快捷键解决？

    如果你的远程 `"+p` 还没修好，但你又想解决每次右键粘贴都要手动打 `:set paste` 的痛苦，你可以使用 **Vim 内置的简写切换** 或者 **切换键**。

    #### 方案 A：一键开关粘贴模式（最常用）

    在 `~/.vimrc` 中设置一个临时切换粘贴状态的快捷键（比如 `F10`）：

    ```vim
    set pastetoggle=<F10>

    ```

    * **操作流**：想粘贴时按一下 `F10`（状态栏出现 `-- INSERT (paste) --`） $\rightarrow$ 鼠标右键粘贴 $\rightarrow$ 再按一下 `F10` 退出。

    #### 方案 B：利用命令行寄存器映射（不需要 `"+` 寄存器支持）

    如果你非要用 `<Leader>p` 并且直接通过终端模拟器塞入内容，可以利用命令行的 `_` 寄存器或直接模拟：

    ```vim
    nnoremap <Leader>p :set paste<CR>i

    ```

    * 这个映射会在按下 `<Leader>p` 后自动开启粘贴模式并**进入插入模式**，此时你只需要直接点**鼠标右键**，粘贴完后再手动按 `Esc` 退出，Vim 会在退出插入模式时自动利用 `set nopaste` 的联动（如果设置了某些现代终端检测）。

    > 💡 **补充：什么是 `<Leader>` 键？**
    > 如果你没有特意设置过它，Vim 默认的 `<Leader>` 键是 **反斜杠 `\**`。所以上面的映射默认是通过快速按下 `\p` 来触发的。如果你想改成逗号，可以在映射前加上：`let mapleader=","`。

* "+p 似乎不受 :set paste 的影响，始终带有 ^M，但是我又不想用鼠标来辅助完成这个粘贴功能，也不想手动输入 ctrl + v

    这是一个非常精准的痛点。你观察得完全没错：**`:set paste` 只能影响你通过“终端鼠标右键”或“软件外壳”强行灌入的文本流，它对 Vim 内部的 `"+p` 寄存器粘贴是完全无效的。** 因为 `"+p` 是 Vim 直接去向 X11 剪贴板要数据，拿过来是什么样就什么样。既然数据源（Windows 复制的内容）自带 `^M`，`"+p` 就会原封不动地把 `^M` 吐出来。

    既然你**不想用鼠标，也不想手动按 `Ctrl+V**`，最优雅的解决办法就是**让 Vim 自行在内存中把 `^M` 过滤掉，然后再粘贴出来**。

    ---

    ### 终极解决方案：用 Vim 脚本自动清洗 `^M`

    我们可以写一个简单的 Vim 函数，把它绑定到你的 `<Leader>p` 上。
    这个函数会做三件事：

    1. 读取 `"+` 寄存器（Windows 剪贴板）里的文本。
    2. 用代码自动把里面的 `\r`（`^M`）全部替换为空。
    3. 把清洗干净的文本贴进文档。

    请把以下代码复制到你的 `~/.vimrc` 中：

    ```vim
    function! CleanAndPaste()
        " 1. 获取系统剪贴板的内容
        let l:clip = getreg('+')
        
        " 2. 将 \r\n (Windows换行) 替换为 \n (Unix换行)，从而移除 ^M
        let l:clean_clip = substitute(l:clip, '\r\n', '\n', 'g')
        " 兜底：万一有单独的 \r 也去掉
        let l:clean_clip = substitute(l:clean_clip, '\r', '', 'g')
        
        " 3. 将清洗后的文本暂存到无名寄存器 并 粘贴出来
        call setreg('"', l:clean_clip, getregtype('+'))
        normal! ""p
    endfunction

    " 映射到你想要的 <Leader>p
    nnoremap <Leader>p :call CleanAndPaste()<CR>

    ```

    ---

    ### 为什么这个方案完美符合你的需求？

    * **纯键盘流**：你只需要在普通模式下按下 `<Leader>p`（默认是 `\p`），没有任何鼠标操作。
    * **无感知清洗**：它在后台瞬间完成 `^M` 的剔除，你不需要手动去打 `:%s/\r//g` 或者敲 `Ctrl+V`。
    * **不影响原剪贴板**：它只在 Vim 粘贴的瞬间做过滤，不会破坏你 Windows 剪贴板里原有的内容。

    你现在可以保存 `.vimrc` 并重新打开文件（或输入 `:source ~/.vimrc`），直接用键盘测试一下效果了！

* 这个方案还得再改一下，只替换每行行尾的 \r\n -> \n。因为如果我在行文中讨论 \r\n 的区别，可能 会被 误杀

    想得太周全了！在行文中讨论技术细节（比如写代码、写文档解释 `\r\n`）时，如果把正文里的 `\r` 也误杀了，确实会造成灾难。

    为了**只精准切掉每行行尾的 `^M`（Windows 换行符的物理残留）**，我们需要对匹配规则进行严格的限定。

    在 Vim 脚本中，系统剪贴板拿到的数据是一个由 `\n` 分隔的字符串。当 Windows 文本传过来时，它的结构其实是 `文本\r\n文本\r\n`。因此，我们只需要**精确匹配在 `\n`（换行）或字符串末尾之前的 `\r**` 即可。

    请将 `~/.vimrc` 中的函数修改为以下版本：

    ```vim
    function! CleanAndPaste()
        " 1. 获取系统剪贴板的内容
        let l:clip = getreg('+')
        
        " 2. 仅替换紧跟在换行符 \n 之前、或者处于整个字符串最末尾的 \r
        "    \ze\n 表示匹配 \r 但只定位在 \n 之前；\%$ 表示字符串绝对末尾
        let l:clean_clip = substitute(l:clip, '\r\ze\n\|\r\%$', '', 'g')
        
        " 3. 将清洗后的文本暂存到无名寄存器并粘贴
        call setreg('"', l:clean_clip, getregtype('+'))
        normal! ""p
    endfunction

    " 映射到 <Leader>p
    nnoremap <Leader>p :call CleanAndPaste()<CR>

    ```

    ### 🔍 这里的正则表达式是如何做到不误杀的？

    核心在于 `\r\ze\n\|\r\%$` 这段正则：

    * **`\r\ze\n`**：匹配 `\r`，但只有当它后面**紧跟著** `\n`（换行符）时才匹配。这里的 `\ze` 是 Vim 正则中的“结尾标记”，意思是“只把 `\ze` 前面的 `\r` 替换掉，后面的 `\n` 保留”。
    * **`\|`**：逻辑“或”（OR）。
    * **`\r\%$`**：匹配处于整个剪贴板文本**最末尾**的那单一个 `\r`（防止最后一行没有换行符却带有一个 `\r`）。

    ### 效果测试

    如果你复制了一段这样的 Windows 文本：

    ```text
    今天我们来讨论 \r\n 的区别。[换行]
    Windows 使用 \r\n，而 Linux 使用 \n。[换行]

    ```

    使用这个新函数粘贴后：

    * 行文中间的 `\r\n` 字符串会**原封不动地保留**。
    * 每行末尾隐藏的、导致格式错乱的物理 `^M` 会被**精准剔除**。

* 可以把整个过程压缩成一行映射命令吗，比如 <leader>p  `<,`>s/\r\n$/\n"+p 之类的

    你非常有想法！确实可以用一行映射来解决，但不能用你写的 `s/\r\n$/\n`，因为 `"+p` 粘贴动作发生时，内容已经落盘到缓冲区了，这时候再去针对“行”做替换，很容易把正文行文中的 `\r\n` 再次误杀。

    既然想压成一行，最优雅的做法是**直接在一行命令里，用 `substitute()` 函数清洗 `"+` 寄存器本身，然后立刻贴出来**。

    请直接把这一行加入你的 `~/.vimrc`：

    ```vim
    nnoremap <Leader>p :call setreg('"', substitute(getreg('+'), '\r\ze\n\|\r\%$', '', 'g'), getregtype('+'))<CR>""p

    ```

    ### 💡 这一行命令拆解：

    1. **`getreg('+')`**：把 Windows 剪贴板的内容瞬间读入内存。
    2. **`substitute(..., '\r\ze\n\|\r\%$', '', 'g')`**：在内存中精准剥离行尾的 `\r`，完全不触碰行文中间的 `\r\n`。
    3. **`call setreg('"', ..., ...)`**：把清洗干净的文本塞进 Vim 的默认寄存器（`"`）。
    4. **`<CR>""p`**：敲回车执行完清洗，紧接着按下 `""p`（从默认寄存器粘贴）。

    这样既满足了你**压成一行**的极简强迫症，又完美继承了**不误杀正文**的精准度，按 `<Leader>p` 即可一气呵成！

* vim 如何显示 markdown 文件的导航

    在 Vim 中显示 Markdown 文件的导航（大纲/目录结构），最有效的方法是使用**插件**。因为 Vim 本身没有内置的侧边栏大纲功能，但通过社区插件，你可以获得像 IDE 一样丝滑的导航体验。

    以下是目前最流行、最好用的几种解决方案：

    ## 方案一：使用 `vim-markdown-toc`（自动生成目录）

    如果你不想在侧边栏挂一个不断渲染的窗口，而是想直接在 Markdown 文件中**插入一个可以跳转的目录（TOC）**，这是最佳选择。

    * **插件地址**：`mzlogin/vim-markdown-toc`
    * **常用命令**：
    * `:GenTocGFM`：生成 GFM 风格的目录。
    * `:UpdateToc`：手动更新目录（通常保存时会自动更新）。



    > **提示**：生成的目录中，把光标移到某个标题上按 `gd`（Go to Definition）或者如果你配合了其他跳转快捷键，就能直接跳到对应段落。

    ---

    ## 方案二：使用 `Tagbar` 或 `Vista.vim`（侧边栏大纲）

    如果你想要一个像 VS Code 那样在左侧或右侧独立显示的**悬浮/固定大纲栏**，可以使用基于 Ctags 的插件。

    ### 1. Vista.vim（强烈推荐，现代且美观）

    `liuchengxu/vista.vim` 是一个非常现代的大纲插件，支持异步渲染，对 Markdown 的支持极好。

    * **安装要求**：需要安装 `universal-ctags`（通过 `brew install ctags` 或 `sudo apt install ctags`）。
    * **使用方法**：
    * 在 Vim 中输入 `:Vista` 即可打开右侧大纲栏。
    * 在大纲栏中，按 `Enter` 键即可跳转到对应的标题。



    ### 2. Tagbar（老牌经典）

    `majutsushi/tagbar` 是最经典的大纲插件。

    * **配置**：因为 Tagbar 默认对 Markdown 的支持一般，你需要在 `.vimrc` 中加上以下配置来让它识别 Markdown 标题：
    ```vim
    let g:tagbar_type_markdown = {
        \ 'ctagstype' : 'markdown',
        \ 'kinds' : [
            \ 'h:Heading_L1',
            \ 'i:Heading_L2',
            \ 'k:Heading_L3',
            \ 'u:Heading_L4',
            \ 'm:Heading_L5',
            \ 'p:Heading_L6'
        \ ]
    \ }

    ```


    * **使用方法**：输入 `:TagbarToggle` 打开或关闭。

    ## 方案三：使用 `coc-outline`（如果你用了 Coc.nvim）

    如果你是用 `coc.nvim` 作为你的补全和 LSP 引擎，你完全不需要额外安装基于 Ctags 的插件。

    * **安装命令**：在 Vim 中运行 `:CocInstall coc-outline`
    * **使用方法**：输入 `:CocOutline`，就会在侧边栏弹出一个非常精准的 Markdown 目录树。

    ## 方案四：纯原生 Vim 技巧（无需任何插件）

    如果你在没有配置插件的服务器上，想要临时瞅一眼导航，可以利用 Vim 的**折叠（Folding）**或**搜索定位**功能：

    1. **利用折叠看大纲**：
    * 输入 `:set filetype=markdown`（确保开启了 Markdown 语法）。
    * 输入 `:set foldmethod=expr`（或者有些配置里直接按 `zi` 打开折叠）。
    * 这时候所有的章节会被折叠起来，只显示标题，你可以通过 `zo`（打开折叠）和 `zc`（关闭折叠）来当成目录浏览。


    2. **快捷搜索标题**：
    * 输入 `/[#]\+ ` 然后按回车，配合 `n` 或 `N` 在各个标题之间快速跳跃。

* 是否有对  * 无序列表生成大纲的

    一般的导航/大纲插件（如 Vista、Tagbar）都是基于标题（Heading，即 `#`、`##`）来生成的，**默认都不会对无序列表（`*`、`-`）生成大纲**。

    不过，想要实现“把无序列表当成大纲来导航”的需求，Vim 有几种非常巧妙的解决办法：

    ---

    ## 方案一：使用原生 `Ctrl-P` 模糊搜索（免插件，最快）

    如果你想在当前文件的所有无序列表中快速跳转，不需要安装任何大纲插件，直接利用 Vim 的搜索缓冲区配合快捷键就能搞定：

    1. 在 Vim 中输入以下命令（可以临时列出所有无序列表并让你选择跳转）：
    ```vim
    :ilist /^\s*[\*+-]\s/

    ```


    2. **更高效的做法**：直接在输入模式或正常模式下，按下快捷键 `/`，输入 `^\s*[*+-] `，然后配合 `n` 或 `N` 在无序列表之间像跳表单一样切换。

    ---

    ## 方案二：利用 Vim 的折叠功能（Folding）

    无序列表通常有嵌套结构（缩进）。你可以利用 Vim 的缩进折叠，把列表以外的内容过滤掉，只留下缩进大纲。

    在你的 `.vimrc` 中加入或者临时输入：

    ```vim
    :set foldmethod=indent

    ```

    * 此时按 `zM` 可以折叠所有有缩进的列表。
    * 整个文件会变成一个由无序列表构成的“可折叠大纲”。
    * 移动到你想去的列表项上，按 `zo` 展开，`zc` 折叠。

    ---

    ## 方案三：如果你使用的是 `CtrlP` 插件

    如果你装了经典的模糊搜索插件 `ctrlp.vim`，它有一个内置的 `crypto-line` 功能，可以通过自定义正则来把无序列表抽出来做成一个弹窗导航。

    在 `.vimrc` 中配置一个快捷键（比如 `<leader>l`）：

    ```vim
    nnoremap <leader>l :CtrlPLine<CR>^\s*[\*+-]\s

    ```

    按下快捷键后，会弹出一个动态模糊搜索框，**里面只实时显示你的无序列表行**，上下键选择，回车直接跳转。

    ---

    ## 方案四：如果你使用的是 `Telescope` (Neovim 用户)

    如果你使用的是 Neovim 并装了 `Telescope`，那就简单太多了。你可以直接利用 `live_grep` 或 `current_buffer_fuzzy_find` 传入正则。

    可以绑定一个快捷键：

    ```lua
    vim.keymap.set('n', '<leader>gl', function()
      require('telescope.builtin').current_buffer_fuzzy_find({ default_text = "^\\s*[\\*+-]\\s" })
    end, { desc = "无序列表大纲" })

    ```

    这会在下方弹出一个非常优雅的浮动窗，里面全是你的无序列表，支持模糊过滤和实时预览跳转。

    ---

    **总结**：由于插件作者通常认为“标题”才算大纲，因此想要给“无序列表”做导航，最完美的体验是**方案三（Vim 用户）**或**方案四（Neovim 用户）**，通过模糊搜索插件过滤出列表项。你目前使用的 Vim 核心是原生的 Vim 还是 Neovim 呢？
