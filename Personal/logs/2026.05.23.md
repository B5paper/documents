* 详细讲讲 PRAGMA table_info('你的表名'); 返回的数据的含义

    当你在 SQLite 中执行 `PRAGMA table_info('你的表名');` 时，它会返回一个表格，详细列出该表每一列（字段）的配置信息。

    通常这个返回的表格会固定包含 **6 列** 数据，它们分别是：`cid`、`name`、`type`、`notnull`、`dflt_value`、`pk`。

    为了让你直观地看懂，我们先假设有一张叫 `users`（用户表）的结构，执行 `PRAGMA table_info('users');` 后，返回的数据大体会长成下面这个样子：

    | cid | name | type | notnull | dflt_value | pk |
    | --- | --- | --- | --- | --- | --- |
    | **0** | id | INTEGER | 0 | *NULL* | 1 |
    | **1** | username | TEXT | 1 | *NULL* | 0 |
    | **2** | age | INT | 0 | 18 | 0 |
    | **3** | status | TEXT | 1 | 'active' | 0 |

    ---

    ### 这 6 列数据的详细含义拆解：

    #### 1. `cid` (Column ID - 列的 ID/序号)

    * **含义**：这一列在表中的排第几。
    * **解读**：从 `0` 开始计数。`0` 代表第一列，`1` 代表第二列，依此类推。它反映了你当初建表（`CREATE TABLE`）时写字段的先后顺序。

    #### 2. `name` (列名)

    * **含义**：字段的名称。
    * **解读**：这就是你最想要的答案！比如上面表格里的 `id`、`username`、`age` 就是每一列的名字。

    #### 3. `type` (数据类型)

    * **含义**：该列声明的数据类型。
    * **解读**：比如 `INTEGER`（整数）、`TEXT`（文本/字符串）、`REAL`（浮点数）、`BLOB`（二进制大数据）等。
    * *补充小知识*：SQLite 采用的是“弱类型（Dynamic Typing）”机制，即便你声明了 `INT`，实际写入字符串有时也是能写进去的，这里显示的只是建表时**声明**的类型。

    #### 4. `notnull` (是否不能为空)

    * **含义**：该列是否带有 `NOT NULL`（非空）约束。
    * **解读**：它是用数字 `0` 和 `1` 来表示布尔值的：
    * **`0`**：代表 **False**。也就是说这一列**允许**为空（允许插入 NULL 值）。
    * **`1`**：代表 **True**。也就是说这一列**不允许**为空（设置了 `NOT NULL`），往里插空数据会报错。



    #### 5. `dflt_value` (Default Value - 默认值)

    * **含义**：如果插入数据时没有指定这一列的值，系统自动填充的默认值。
    * **解读**：
    * 如果显示 *NULL*，代表没有设置默认值。
    * 如果像上面示例中的 `18` 或 `'active'`，就代表当你漏填这个字段时，系统会自动把 18 或 'active' 填进去。



    #### 6. `pk` (Primary Key - 是否为主键)

    * **含义**：这一列是不是表的主键。
    * **解读**：同样用数字来表达：
    * **`0`**：代表**不是**主键。
    * **`1`**：代表**是**主键。
    * *特殊情况*：如果你的表设置了**联合主键**（也就是由多个字段共同组成一个主键），那么是主键的几列都会显示大于 0 的数字（如 `1`, `2`），数字代表它们在联合主键中的先后顺序。
