    ### 1. 为什么会出现负值？

    光密度的计算公式通常遵循：


    $$\Delta OD = -\ln\left(\frac{I_{t}}{I_{0}}\right)$$

    * **$I_t$**：当前的测量强度（Intensity at time $t$）。
    * **$I_0$**：基准强度（Baseline Intensity）。

    在 MNE 的 `optical_density` 函数中，默认情况下它会将**整个时间序列的平均值**（mean）作为基准 $I_0$。

    * 当某个时刻的信号强度 **$I_t$ 大于平均值**时，括号内的比值 $>1$，其自然对数 $\ln$ 为正，前面加个负号，结果就是**负值**。
    * 这通常意味着：此时组织对光的吸收比平均水平更低（例如血红蛋白浓度暂时下降，或者信号向上波动）。

    ---

    ### 2. 负值的物理含义

    在 NIRS 领域，我们关注的通常不是绝对光密度，而是**光密度的变化量（Changes in OD）**。

    * **负值（Negative $\Delta OD$）**：表示相对于基准线，光吸收减少了。
    * **正值（Positive $\Delta OD$）**：表示相对于基准线，光吸收增加了。

    如果你后续要运行 `beer_lambert_law`（修正巴比定律），这些正负值会直接转化为血红蛋白浓度的相对变化（$\Delta HbO$ / $\Delta HbR$）。

    ---

    ### 3. 什么情况下负值是不正常的？

    虽然负值本身没问题，但如果出现以下情况，你需要检查原始数据：

    1. **原始强度（Intensity）本身有负值**：
    在执行 `optical_density` 之前的 `raw` 数据（`fnirs_cw_amplitude`）中，如果由于硬件偏置（Offset）或环境光干扰出现了负数，那么 $\ln$ 计算会失效或产生异常。你可以检查：
    `print(np.any(raw.get_data() <= 0))`
    2. **数值极其巨大（如 -100 或 100）**：

    这通常是因为你的 **Source-Detector 距离（SD distance）** 被识别为 **0**。

    在 `beer_lambert_law` 的计算公式中，浓度变化 $\Delta C$ 与发射端到接收端的距离 $L$ 成反比。如果 MNE 找不到通道的坐标信息，它会默认距离 $L = 0$。为了避免除以零的错误（Divide by zero），内部计算可能会导致结果被置为 0。

    ### 核心原因：缺少传感器坐标

    你之前是手动创建的 `RawArray`。如果你只是设置了 `ch_names` 和 `ch_types`，但**没有为 Source（发射端）和 Detector（接收端）设置 3D 坐标**，MNE 就无法计算它们之间的欧几里得距离。

    没有距离，Beer-Lambert Law 就无法将光密度变化转换为浓度。

    ---

    ### 解决方案：手动设置坐标

    你需要为每个通道定义 Source 和 Detector 的位置。对于 NIRS 通道，MNE 存储坐标的位置在 `chs[i]['loc']`：

    * `loc[3:6]`：Source 的 X, Y, Z 坐标。
    * `loc[6:9]`：Detector 的 X, Y, Z 坐标。

    #### 快速修正示例：

    如果你只是想跑通流程，可以手动给一个标准的距离（比如 3cm，即 0.03m）：

    ```python
    # 假设你的 SD 距离都是 30mm (0.03m)
    for i in range(len(raw_od.ch_names)):
        # 设置 Source 坐标 (0, 0, 0)
        raw_od.info['chs'][i]['loc'][3:6] = [0, 0, 0]
        # 设置 Detector 坐标 (0.03, 0, 0) -> 这样距离就是 0.03
        raw_od.info['chs'][i]['loc'][6:9] = [0.03, 0, 0]

    # 现在重新运行转换
    raw_haemo = beer_lambert_law(raw_od, ppf=0.1)
    print(raw_haemo.get_data()[:, 0]) # 检查第一列，应该不再是 0 了

    ```

    ### 为什么会出现这种情况？（排查清单）

    1. **距离确实为 0**：
    你可以运行以下代码检查 MNE 计算出的距离：
    ```python
    from mne_nirs.channels import get_source_detector_distances
    dist = get_source_detector_distances(raw_od.info)
    print(dist) # 如果输出全是 0，就是这里的问题

    ```


    *(注意：`get_source_detector_distances` 可能需要安装 `mne-nirs` 库)*
    2. **单位问题**：
    MNE 的标准单位是**米 (meters)**。如果你填写的坐标单位是毫米且数值很小（比如 `0.0003`），计算出的浓度可能会因为极小而被四舍五入显示为 0。
    3. **PPF/DPF 设置**：
    虽然你设置了 `ppf=0.1`，但如果距离 $L=0$，公式 $\Delta C = \frac{\Delta OD}{\epsilon \cdot L \cdot DPF}$ 的分母就变成了 0。
