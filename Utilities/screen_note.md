# screen note

## cache

* 使用 screen 后，进入 ctrl + a, [ 的复制模式，然后使用两次空格复制完数据后，进入 vim 中，insert 模式，然后使用 ctrl + a, ] 粘贴。但是由于 vim 有 autoindent，此时会造成输出里有大量的自动缩进。如何解决这个问题？

    这是一个非常经典且让人头疼的“梯形缩进”问题。原因在于 Vim 的 `autoindent`（自动缩进）误把 `screen` 粘贴进来的文本当成了你正在**手动输入**的代码，结果每一行都在前一行的基础上叠加上一次缩进。

    解决这个问题的最佳方案是利用 Vim 的 **`paste` 模式**。在这个模式下，Vim 会暂时关闭所有的自动缩进和快捷键映射，完美还原复制的内容。

    以下是几种最实用的解决方法：

    **方法一：临时开启 paste 模式（最常用）**

    在准备粘贴之前，先在 Vim 中执行以下步骤：

    1. 按 `Esc` 退出当前的 Insert 模式，回到 Normal 模式。
    2. 输入 `:set paste` 并回车。
    3. 按 `i` 进入 Insert 模式（此时底部会显示 `-- INSERT (paste) --`）。
    4. 使用 `Ctrl + a, ]` 粘贴你的文本。
    5. 粘贴完成后，按 `Esc` 回到 Normal 模式，输入 `:set nopaste` 恢复普通的编辑模式。

    **方法二：绑定快捷键快速切换（一劳永逸）**

    频繁输入 `:set paste` 略显麻烦，你可以在你的 `~/.vimrc` 文件中加一行配置，绑定一个快捷键（比如 `F2`）来一键切换：

    ```vim
    set pastetoggle=<F2>
    ```

    **使用方法：**

    在 Vim 的 Insert 模式或 Normal 模式下，直接按下 `F2` 键，就能在“普通模式”和“粘贴模式”之间无缝切换，省去了手动敲命令的麻烦。

    screen 的复制缓冲区与文件的交互：

    * 方案一：让 screen 把复制内容自动写进文件，Vim 直接读取

        这是纯终端环境下的终极魔改。我们可以让 `screen` 在复制时，把内容同步写到一个临时文件里，然后让 Vim 去读这个文件。

        1. 在你的 `~/.screenrc` 中加入这行配置（允许写出缓冲区）：

            ```screen
            bufferfile /tmp/screen-exchange
            ```

        2. 在 `~/.vimrc` 中写一个快捷键（比如映射到 `F3`）：

            ```vim
            nnoremap <F3> :r /tmp/screen-exchange<CR>
            ```

        **使用流程：**

        * 在 `screen` 中用 `Ctrl + a, [` 选中并复制文本。
        * 复制完成后，按下 `Ctrl + a, >`（这会把刚刚复制的内容写入 `/tmp/screen-exchange` 文件）。
        * 打开 Vim，在 **Normal 模式**下直接按 `F3`。Vim 会直接把文件内容读进来，**完全不经过 Insert 模式，因此绝对不会触发任何自动缩进！**

    * 方案二：利用 `xclip` / `xsel` 连通系统剪贴板（需要 X11 支持）

        如果你的 `screen` 运行在本地 Linux，或者通过 `ssh -X` 连接到服务器，你可以让 `screen` 复制的内容直接进入系统的 `+` 寄存器。

        在 `~/.screenrc` 中加入：

        ```screen
        bindkey -m ' ' eval 'stuff \015' 'writebuf' 'exec sh -c "xclip -sel clip /tmp/screen-exchange"'
        ```

        *(注：这串命令的意思是，当你用空格结束 screen 复制时，它自动把内容扔给系统的剪贴板 `xclip`)*

        一旦配置成功，`screen` 复制的内容就真的进入系统剪贴板了。此时你在 Vim 中（需要 Vim 支持 `+clipboard`），直接在 Normal 模式下敲 **`"+p`** 就能秒贴，同样不需要进入 Insert 模式，完美避开缩进问题。

    **总结来说：** 如果不想折腾复杂的配置，遇到 `screen` 粘贴时，最省心的方式依然是前面提到的 **方法二（用 `F2` 切换 `pastetoggle`）**。

    * screen append

        默认情况下，如果你直接使用 `Ctrl + a, >`（执行的是 `writebuf` 命令），`screen` 会**直接覆盖**旧的 `screen-exchange` 文件，而不会追加。

        但是，`screen` 内部其实提供了一个**内置的追加机制**。它允许你在**复制文本的阶段**就把内容“追加”到 `screen` 的内部缓冲区里，然后再通过 `Ctrl + a, >` 一次性写出去。

        这里有两种实现追加的方法：

        方法一：利用 Screen 的内置“追加复制”模式（推荐，纯原生）

        当你进入 `screen` 的复制模式后，可以通过以下操作实现追加：

        1. 按 `Ctrl + a, [` 进入复制模式。
        2. 移动光标到你想复制的起点，按下**第一下空格**（设置起点）。
        3. 移动光标到终点。**注意：此时不要按空格，而是按下大写的 `A` 键。**
        4. 按下 `A` 后，当前选中的文本就会追加（Append）到你上一次复制的内容后面，并自动退出复制模式。
        5. 此时再按下 `Ctrl + a, >`，写入到 `screen-exchange` 文件里的就是多次累加后的完整内容了。

        方法二：在 Shell 层面通过重定向追加（适合需要多次独立写入）

        如果你希望每次按 `Ctrl + a, >` 时，它在文件末尾追加，而不是覆盖，原生 `writebuf` 命令没有直接提供 `append` 参数。但我们可以通过在 `~/.screenrc` 中自定义一个快捷键，利用 shell 的 `>>` 符号强行追加。

        在你的 `~/.screenrc` 中加入这样一行：

        ```screen
        # 将 Ctrl + a, M 绑定为：将缓冲区写到临时文件，然后利用 cat 追加到 screen-exchange
        bind M eval "writebuf /tmp/screen-tmp" "exec sh -c 'cat /tmp/screen-tmp >> /tmp/screen-exchange'"
        ```

        **使用流程：**

        1. 正常用 `Ctrl + a, [` 和双击空格复制第一段。
        2. 按下 `Ctrl + a, M`（大写 M），内容被追加到 `/tmp/screen-exchange`。
        3. 复制第二段，再次按下 `Ctrl + a, M`……
        4. 这样你就可以无限追加。最后在 Vim 中直接用 `:r /tmp/screen-exchange` 一把读入。
        *(注：用这个方法前，记得先去 `/tmp/screen-exchange` 把旧文件删掉或清空，以免混入以前的历史数据。)*

* screen 如何将默认 shell 设置成 bash?

    在 `screen` 中，它默认通常会调用你的系统默认 Shell（可以通过 `echo $SHELL` 查看）。如果你想强制让 `screen` 启动时使用 **bash**，可以通过以下两种方式实现：

    **方法一：修改配置文件（永久生效，推荐）**

    这是最标准的方法。通过修改当前用户的 `~/.screenrc` 文件，指定 `shell` 参数。

    1.  打开（或创建）配置文件：
        ```bash
        vim ~/.screenrc
        ```
    2.  在文件中添加以下一行：
        ```text
        shell /bin/bash
        ```
        *如果你希望启动的是 **Login Shell**（这样会加载 `~/.bash_profile` 等配置），可以写成：*
        ```text
        shell -/bin/bash
        ```
        *(注意前面的横杠 `-`)*

    ---

    **方法二：启动时临时指定**

    如果你不想修改配置文件，只是偶尔想在 `screen` 里用 bash，可以在启动命令后直接加参数：

    ```bash
    screen /bin/bash
    ```

    ---

    **方法三：修改系统的默认 Shell（全局影响）**

    如果你希望无论是在 SSH 登录还是在 `screen` 里都默认使用 bash，可以直接修改账户的默认设置：

    ```bash
    chsh -s /bin/bash
    ```
    > **注意**：执行此命令后通常需要重新登录终端才能生效。

    ---

    **验证是否生效**

    进入 `screen` 后，你可以运行以下命令来确认当前的 Shell：

    ```bash
    echo $0
    ```
    如果输出是 `-bash` 或 `/bin/bash`，说明设置成功了。

    ---

    **小贴士：**

    如果你发现进入 `screen` 后 bash 的**颜色消失了**或者 **alias 失效了**，通常是因为 `screen` 默认启动的是非登录 Shell（Non-login shell）。此时建议使用方法一中的 `shell -/bin/bash`（带横杠的形式），这样它就会像你刚登录系统时一样，完整加载所有的环境变量。

    你是遇到了进入 `screen` 后环境变量没加载的问题，还是单纯想把默认的 `sh` 或 `zsh` 换掉？

* 我如何判断一个 terminal 窗口是否开了 screen ?

    有几种方法可以判断当前终端是否运行在 `screen` 会话中：

    1. 检查环境变量（最常用）

        ```bash
        echo $STY
        ```

        - **有输出**（如 `1234.tty1`）：在 screen 中
        - **空输出**：不在 screen 中

        `STY` 是 screen 会话特有的环境变量，记录当前会话名称。

        ```bash
        # 一行判断并输出结果
        [ -n "$STY" ] && echo "在 screen 中" || echo "不在 screen 中"
        ```

    2. 检查 TERM 变量

        ```bash
        echo $TERM
        ```

        - **screen 中**：通常显示 `screen` 或 `screen-256color`
        - **普通终端**：显示 `xterm`、`xterm-256color`、`linux` 等

        但这个方法**不够准确**，因为可以手动修改 TERM 变量。

    3. 查看父进程

        ```bash
        ps -p $PPID -o comm=
        ```
        - 输出 `screen`：在 screen 中
        - 输出其他（如 `bash`、`zsh`、`sshd`）：不在 screen 中

        更详细的父进程信息：
        ```bash
        ps -f -p $PPID
        ```

    4. 检查进程树

        ```bash
        pstree -p $$ | grep screen
        ```
        或
        ```bash
        ps -ef | grep -v grep | grep "screen"
        ```

    5. 检查终端内的按键行为

        在 screen 中，默认的转义键是 `Ctrl+A`：
        - 按 `Ctrl+A` 后按 `?`：会显示 screen 帮助（如果在 screen 中）
        - 按 `Ctrl+A` 后按 `d`：会脱离（detach）screen 会话

        **注意**：这个方法最直观，但会触发 screen 命令，可能产生副作用。

    **推荐判断方法**

    **最可靠**的方法是结合环境变量和父进程：

    ```bash
    if [ -n "$STY" ]; then
        echo "在 screen 会话中: $STY"
    elif ps -p $PPID -o comm= | grep -q screen; then
        echo "父进程是 screen"
    else
        echo "不在 screen 中"
    fi
    ```

    或者创建一个别名方便使用：

    ```bash
    alias isscreen='[ -n "$STY" ] && echo "In screen: $STY" || echo "Not in screen"'
    ```

    **类似工具的环境变量**

    其他终端复用工具也有类似的环境变量：
    - **tmux**：`echo $TMUX`（有值表示在 tmux 中）
    - **screen**：`echo $STY`（有值表示在 screen 中）

* 简述 screen 的用法

    `screen` 是一个 **终端复用器**。它最核心的价值在于：即使你断开了 SSH 连接，原本在服务器上运行的任务也不会中断，你可以随时“重连”回去查看进度。

    安装：`sudo apt install screen`

    以下是 `screen` 的常用操作指南：

    1. 核心操作流程

        通常的使用逻辑分为三步：**创建 -> 挂起（离开）-> 恢复**。

        | 操作步骤 | 命令 / 快捷键 | 说明 |
        | --- | --- | --- |
        | **新建窗口** | `screen -S <名字>` | 创建并进入一个自定义命名的会话 |
        | **离会话** | `Ctrl + a`, 然后按 `d` | **保持任务运行**，并退回到主终端 (Detach) |
        | **查看列表** | `screen -ls` | 查看当前后台运行的所有会话及其 ID |
        | **恢复会话** | `screen -r <名字或ID>` | 重新进入之前挂起的会话 (Resume) |
        | **彻底关闭** | `exit` 或 `Ctrl + d` | 在会话内输入，任务结束并销毁该会话 |

    2. 常用快捷键（在会话内部使用）

        所有 `screen` 指令都以 `Ctrl + a` (通常简写为 `C-a`) 作为前缀。

        * `Ctrl + a` + `d`：脱离当前会话（最常用）。

        * `Ctrl + a` + `k`：强制杀掉当前窗口。

        * `Ctrl + a` + `[`：进入**复制模式**。你可以用方向键翻页查看历史滚屏记录，按 `Esc` 退出。

        * `Ctrl + a` + `c`：在当前会话中创建一个新的子窗口。

        * `Ctrl + a` + `n / p`：在多个子窗口间切换（Next / Previous）。

    3. 进阶技巧

        * **清理无效会话**：如果程序崩溃导致会话显示为 `Dead`，使用 `screen -wipe` 清除。

        * **强制接管**：如果会话显示 `Attached`（可能在另一台电脑连着），用 `screen -d -r <名字>` 强制将其在那头断开并在此处恢复。

        * **日志记录**：使用 `screen -L` 启动会话，可以将所有输出自动保存到日志文件中。

    常见场景示例

    如果你要运行一个耗时 10 小时的脚本：

    1. 输入 `screen -S train_model`。

    2. 在新窗口里执行 `./script.sh`。

    3. 按 `Ctrl + a`, `d` 潇洒下班，关掉笔记本。

    4. 第二天回公司，输入 `screen -r train_model` 查看结果。

* 如何查看当前会话中子窗口的信息

    在 `screen` 的单个会话中，如果你通过 `Ctrl + a` + `c` 创建了多个子窗口（Window），可以通过以下几种方式查看这些窗口的信息：

    1. 调出窗口列表（最直观）

        按下快捷键：

        > **`Ctrl + a`** 然后按 **`"`** (双引号键)

        你会看到一个交互式的列表，列出了当前会话中所有活跃的子窗口。

        * **操作**：使用方向键上下移动，按 **Enter** 键即可切换到选中的窗口。
        * **标识**：列表里带有 `*` 的是当前正在查看的窗口，带有 `-` 的是上一个切换过的窗口。

    2. 状态栏底部查看

        按下快捷键：

        > **`Ctrl + a`** 然后按 **`w`**

        屏幕的左下角（或状态栏）会弹出一行临时文字，显示类似 `0* bash  1- bash  2 bash` 的信息：

        * **数字**：窗口编号。
        * **星号 `***`：代表你当前所在的窗口。
        * **减号 `-**`：代表上一个窗口。
        * **名称**：默认通常是你的 Shell 名称（如 `bash` 或 `zsh`）。

    3. 查看当前窗口的详细状态

        按下快捷键：

        > **`Ctrl + a`** 然后按 **`i`** (info)

        这会在屏幕底部显示当前窗口的详细参数，包括：

        * 窗口的分辨率（宽x高）。
        * 当前使用的字符编码（如 `UTF-8`）。
        * 窗口编号和名称。

    💡 进阶小贴士：给窗口起个名

    当子窗口多了以后，全叫 `bash` 会很难分辨。你可以给当前所在的窗口重命名：

    1. 按下 **`Ctrl + a`** 然后按 **`A`** (大写 A，即 `Shift + a`)。
    2. 删掉旧名字，输入新名字（如 `Database` 或 `Log-Monitor`）并回车。
    3. 之后再用 `Ctrl + a` + `"` 查看时，就能一眼认出哪个窗口是做什么的了。

* screen 在子窗口间切换

    如果你创建了多个子窗口（Window 0, 1, 2...），可以使用以下快捷键：

    * **快速轮换**：

        * `Ctrl + a` 然后按 `n` (Next)：切换到下一个窗口。

        * `Ctrl + a` 然后按 `p` (Previous)：切换到上一个窗口。

    * **按编号跳跃**：

        * `Ctrl + a` 然后按 **数字键 (0-9)**：直接跳到对应的窗口。

    * **回看切换**：

        * `Ctrl + a` 然后按 `Ctrl + a`：在当前和上一次查看的两个窗口之间快速来回切换。

* screen 分栏显示（类似 Vim 的 Split）

    `screen` 确实支持分栏，但逻辑和 Vim 略有不同：它先切分“区域（Region）”，然后你需要把“窗口（Window）”填进去。

    分栏操作：

    * 水平分栏：`Ctrl + a` 然后按 `S` (大写 S)。

    * 垂直分栏：`Ctrl + a` 然后按 `|` (竖线键，通常是 `Shift + \`)。

    分栏后的导航：

    * 移动光标：
    
        `Ctrl + a` 然后按 `Tab`。光标会在不同的分栏区域间跳动。

    * 填入内容：

        刚分好的栏是**空白**的。你需要跳到那个空白区域，然后：

        * 按 `Ctrl + a` 然后按 `c`：在那里创建一个新窗口。

        * 或者按 `Ctrl + a` 然后按数字（如 `0`）：把已有的窗口调到这个区域显示。

    关闭分栏：

    * **关闭当前栏**：`Ctrl + a` 然后按 `X` (大写 X)。

    * **只留当前栏（关闭其他所有）**：`Ctrl + a` 然后按 `Q` (大写 Q)。

    **Screen vs Tmux：一个真诚的建议**

    虽然你在问 `screen`，但不得不提的是，如果你对**分栏**、**鼠标拖拽**、**持久化布局**有较高要求，**Tmux** 是目前更现代、更强大的选择。

    | 特性 | Screen | Tmux |
    | --- | --- | --- |
    | **垂直分栏** | 部分老版本不支持 | 原生完美支持 |
    | **分栏持久化** | 重连后布局常消失 | 配合插件可完美恢复布局 |
    | **状态栏** | 需复杂配置 | 默认自带，信息丰富 |
    | **学习曲线** | 略低 | 略高（但回报大） |

    **快速上手分栏练习：**

    你可以现在尝试：

    1. `Ctrl + a` -> `S`（屏幕一分为二）。
    2. `Ctrl + a` -> `Tab`（跳到下半部）。
    3. `Ctrl + a` -> `c`（下半部出现新命令行）。

* screen 如何上下滚动查看 terminal 输出

    在 `screen` 中，你不能直接用鼠标滚轮或平常的 `Shift + PageUp/PageDown` 来滚动屏幕，因为 `screen` 接管了终端的显示缓存。

    要查看历史输出，你需要进入 `screen` 的 **“复制模式”（Copy Mode）**。

    1. 核心操作步骤

        1. **进入模式**：按下 **`Ctrl + a`**，然后按 **`[`** (左方括号)。

            * 此时你会发现光标变成了一个可以移动的方块，屏幕右上角可能会出现类似 `Copy mode` 的提示。

        2. **上下滚动**：

            * **方向键**：上/下/左/右 逐行移动。
            * **翻页键**：`PageUp` 和 `PageDown`。
            * **类 Vim 快捷键**（推荐）：

                * `k` / `j`：上 / 下移一行。
                * `u` / `d`：向上 / 向下翻半屏 (Up / Down)。
                * `b` / `f`：向上 / 向下翻全屏 (Back / Forward)。
                * `g`：跳到文件最开头；`G`：跳到文件最末尾。

        3. **退出模式**：按 **`Esc`** 键（连按两次更保险）或 `Enter`。

            * 退出后，光标会回到最底部，恢复正常的命令行输入状态。

    2. 为什么我滚不上去？（缓存设置）

        如果进入模式后发现只能向上翻几行就到头了，说明 `screen` 默认的**回滚缓冲区（Scrollback Buffer）**太小了。

        **临时解决方法**：

        启动 screen 时指定缓冲区行数（例如 10000 行）：

        ```bash
        screen -h 10000
        ```

        **永久解决方法**：

        修改你的 `~/.screenrc` 文件（如果没有就创建一个）：

        ```bash
        nano ~/.screenrc
        ```

        在文件中添加这一行：

        ```bash
        defscrollback 10000
        ```

        保存退出。以后新建的每一个 screen 窗口都会拥有 10000 行的历史记录。

    3. 进阶技巧：搜索关键词

        进入 `Copy mode` (`Ctrl + a` + `[`) 后，你可以像在 `less` 或 `vim` 中一样搜索内容：

        * 按 **`/`**：向下搜索字符串。
        * 按 **`?`**：向上搜索字符串。
        * 按 **`n`**：跳转到下一个匹配项。

    **提示**

    如果你使用的是 **Tmux**，操作逻辑非常相似：`Ctrl + b` + `[` 进入模式，然后可以用方向键或 Vim 键位滚动。

* 那么进入 copy 模式后，我该如何使得键盘像vim 那样复制内容？

    既然你已经进入了 **Copy Mode** (`Ctrl + a` + `[`），`screen` 本身的设计逻辑就非常接近经典的 `vi`。

    以下是在 `screen` 中像 Vim 一样“选取、复制、粘贴”的标准动作：

    1. 像 Vim 一样复制 (Copy)

        在 Copy Mode 状态下（光标可自由移动时）：

        1. **定位起始点**：使用 `h/j/k/l` 或方向键将光标移动到你想复制的文字开头。
        2. **标记开始 (Set Mark)**：按下 **`Space`** (空格键)。

            * 你会发现光标下的字符被反色显示，这表示选区开始了。


        3. **移动选区**：继续移动光标，你会看到被选中的部分变色。

        4. **完成复制 (Yank)**：再次按下 **`Space`** (空格键) 或 **`Enter`**。
            * 选中的内容现在已经存入 `screen` 的内部剪贴板（Exchange Buffer）中。

            * 此时会自动退出 Copy Mode，回到正常终端状态。

    2. 像 Vim 一样粘贴 (Paste)

        要把刚才复制的内容贴出来：

        * 按下快捷键：**`Ctrl + a`** 然后按 **`]`** (右方括号)。

            * 这就相当于 Vim 里的 `p`。

    3. 高级技巧：Vim 党必备

        如果你习惯了 Vim 的快捷键，在 Copy Mode 里这些也是通用的：

        * **`0` / `$**`：跳到行首 / 行尾。
        * **`w` / `b**`：按单词向后 / 向前跳。
        * **`g` / `G**`：跳到缓冲区最顶端 / 最底端。
        * **`/`**：向下搜索；**`?`**：向上搜索。

    ⚠️ 一个常见的“坑”：剪贴板隔离

    **请注意**：`screen` 的这个复制粘贴是**它内部的**。

    * 这意味着你用 `Space` 复制的内容，**无法直接**在你的 Windows 或 macOS 桌面通过 `Ctrl + V` 粘贴出来。

    * 它只能在同一个 `screen` 会话的其他窗口（Window）里通过 `Ctrl + a` + `]` 粘贴。

    > **如果你想把服务器上的内容复制到本地电脑：**

    > 建议按住键盘上的 **`Shift`** 键，然后直接用**鼠标拖拽选中**（这样会绕过 `screen` 的拦截，调用本地终端模拟器的复制功能），再在本地执行常规粘贴。
